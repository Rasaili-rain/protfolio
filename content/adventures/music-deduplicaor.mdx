Next we make a struct 
```rust
struct Music {
    artist: String,
    title: String,
    album: String,
    path: PathBuf,
}
```
so that it is easier to pass around the music


Now i wanna make a hashmap of the music to count
``` rust 
let mut music_count: HashMap<Music, usize> = HashMap::new();

*music_count.entry(music).or_insert(0) += 1;

```


but as you might expect for this to work propely we need to remove the pathBuf from the Music struct. As if the path were to stay there would never be a hashMap Collison

to solve this lets take a step back and remove the path from te Music struct

but losing the path would really suck so instead of makina a path a part of the key in a hashmap , lets make multiple paths values of the hashmap

like so
```rust
let mut music_map: HashMap<Music, Vec<PathBuf>> = HashMap::new();
```

and we can populate the map as so
```rust
music_map
    .entry(music)
    .or_insert_with(Vec::new)
    .push(path.to_path_buf());
```
